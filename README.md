Vehicle Management System - Spring DI ProjectThis repository contains a Java application developed using Spring Framework 4.2.2, designed to demonstrate Object-Oriented Programming (OOP) inheritance and various Dependency Injection (DI) techniques.The project is structured as an Eclipse project named Spring-Project2 and runs on the JavaSE-21 environment.📋 Project RequirementsThe application implements a vehicle hierarchy and manages object instantiation via Spring according to the following specifications:Base Class: Vehicle (Properties: price, maximum speed).Derived Classes:Autovehicle: Adds brand and year of fabrication.Motorcycle: Adds type (e.g., classic, speed, off-road).Truck: Adds maximum load.Implementation Rules: Each class includes private member variables, default and parameterized constructors, getters/setters, and a toString() method.Objective: Create and display 5 objects of different types where the MainApp class depends only on the Vehicle base class, not the derived types.⚙️ Dependency Injection MethodsBased on the beans_proiect.xml configuration, the project demonstrates three distinct injection methods:1. XML Setter InjectionUsed for Autovehicle objects (auto1, auto2).Properties like brand, year, and price are injected using the <property> tag.2. XML Constructor InjectionUsed for Truck objects (truck1, truck2).Values for price, maxSpeed, and maxLoad are passed via the <constructor-arg> tag.3. Annotation-Based InjectionUsed for the third type of vehicle (Motorcycle).The project uses <context:component-scan base-package="proiect2" /> to detect classes marked with @Component and inject values using @Value.🛠️ Technology StackLanguage: Java 21 (JavaSE-21).Framework: Spring 4.2.2 (Modules: core, beans, context, jdbc, tx, aop).Database Connector: MySQL Connector 5.1.20 (included in classpath).IDE: Eclipse.🚀 How to RunClone the repository: Ensure you maintain the folder structure.Librariers: Ensure the Spring JAR files and commons-logging are available in your local environment, as the current .classpath points to local absolute paths.Import: Import the project into Eclipse as an existing project.Execute: Run MainApp.java to see the 5 injected vehicle objects printed to the console.
